# LPP Report Template — 报告模板与生成指南

本文档定义LPP报告的输出格式：**双面报告结构**。

## 核心原则

报告分两部分输出，面向不同阅读需求：

```
第一部分（心理画像）        第二部分（评分与解释）
 感性 · 可读性优先           理性 · 分析性优先
 先让人"感觉对"              后让人"理解为什么"
 不出现技术标记              每一步都可追溯
```

两部分基于**同一套Step 1→Step 3的分析结果**，不矛盾。第一部分写完后的审计方法：逐句检查第一部分的每个描述，在第二部分中都能找到对应的语言模式证据。

## 报告篇幅规则

报告深度与样本量成正比：

| 样本量 | 报告范围 | 说明 |
|--------|---------|------|
| < 100字 | 不分析 | 请求更多样本 |
| 100-300字 | 精简报告 | 仅第一部分，标注⚠️"样本量有限，结论可能不够全面" |
| 300-600字 | 标准报告 | 第一部分 + 第二部分精简版（仅关键模式+人格归纳） |
| 600+字 | 完整报告 | 两部分完整输出 + ND观察层（如有标记激活）+ 可选输出层 |
| 任意字数（拼凑样本） | 拼凑模式报告 | 标注⚠️"该样本由不同时期/对象/情景碎片拼凑而成"，各评分附碎片间离散度标记，依恋仅当跨碎片一致时输出 |

**ND层深度控制**：ND层在**标准报告及以上**时启用。仅当样本中检测到≥1个ND标记激活（ND1-ND6任一达到中信号以上）时，在报告中输出ND层。未检测到则不输出。

## 第一部分：心理画像

### 定位

这是用户第一眼看到的内容。目标是让读者在读完后说"这说的是我"，而不是"这是一堆分析术语"。

### 写作框架

以叙事性语言呈现一个人，聚焦整体感而非碎片输出。

**三步内部思维路径**（不呈现在报告中）：
1. **从Step 3取核心判断**：最高/最低的人格维度是什么？依恋的核心矛盾是什么？
2. **从Step 1取最具表现力的语言模式**：哪个模式最能说明上述判断？找那个有画面感的例子。
3. **写出来，删掉所有技术术语**。

**写作规则**：

1. **形式模式驱动**：每一句描述的底牌都是重复出现的语言形式（句式结构/标点习惯/语法偏好），而非叙述内容（"你经历了什么"）。不要在表层提到"你说因为所以所以你很理性"——而是直接说"你习惯用理解来消化情绪"，依据是因果链密度翻倍的观察，但依据不出现在这一层。

2. **硬币两面**：每个特征都呈现优势+代价。"你有一项能力，但它也会让你付出X"——正面→反面自然衔接。

3. **用"你"贯穿**，直接与读者对话。

4. **不出现任何技术标记**：没有D1-D25、没有光谱数值(1-10)、没有评分条(████)、没有大五字母(OCEAN)。这些全部在第二部分出现。

5. **第一条定调**：报告的开头第一句最重要——它给出核心判断，让读者立刻感觉到"这说的是我"。

6. **节奏**：短段落为主。一条特征2-4句。不堆砌。留白。

7. **自然收束**：不要以"总之""综上所述"结尾。以沟通建议和建设性视角收尾，让读者读完后有方向感。

### 叙事结构

```
━━━ [人物] 心理画像 ━━━
```

### 结构化输出骨架

以下为心理画像部分**必须出现的段落**，按顺序排列。每段的实际内容自由发挥，但段落类型和顺序不可跳过或调换。

```
┌─ 段落1：定调句 ──────────────────────────────┐
│ 1-2句，给出核心判断                              │
│ 句式示例：「你是一个[核心特征]的人」              │
│ 要求：读完这句读者想说"这说的是我"                │
└────────────────────────────────────────────────┘
        ↓
┌─ 段落2：最突出的认知/思维特征 ──────────────────┐
│ 2-4句，硬币两面结构                             │
│ 先描述特征（用语言模式为底气但不提模式本身）       │
│ 然后"这让你[正面]，但也让你[反面]"                │
└────────────────────────────────────────────────┘
        ↓
┌─ 段落3：关系/情绪特征 ─────────────────────────┐
│ 2-4句，硬币两面结构                             │
│ 如果样本文本不含关系场景，此段写"文本类型限制"      │
│ 而非编造不存在的关系特征                          │
└────────────────────────────────────────────────┘
        ↓
┌─ 段落4：核心矛盾/张力（如有） ──────────────────┐
│ 1-3句，描述张力如何体现                         │
│ 结尾点明：「这不是矛盾，是同一套加工系统的不同输出」│
│ 如果无显著矛盾，此段跳过                          │
└────────────────────────────────────────────────┘
        ↓
┌─ 段落5：沟通建议 ─────────────────────────────┐
│ 以「跟你相处的人可以知道：」开头                 │
│ 3条建议，每条一句话                             │
│ 建议从语言模式推导而来，不凭空编造               │
└────────────────────────────────────────────────┘
        ↓
┌─ 段落6：建设性视角（收尾） ─────────────────────┐
│ 以「也许可以这样理解你：」或类似自然句式开头       │
│ 1-2句，读完后有方向感                           │
│ 不以"总之/综上所述/总结"结尾                     │
└────────────────────────────────────────────────┘
```

> 定调句 + 认知特征 + 情绪/关系特征 + [可选矛盾] + 沟通建议 + 建设性视角 = 6段骨架。
> 核心特征不足3个时不硬凑，段落2/3缺省时在对应位置注明"文本未提供该维度数据"。

### 写作检查清单

- [ ] 每条描述背后都有对应的形式模式依据（去第二部分验证）？
- [ ] 没有出现D编号/光谱数值/评分条？
- [ ] 第一条能否让读者说"这说的是我"？
- [ ] 特征是否都呈现了正面+反面？
- [ ] 如果读者只读第一部分，是否感觉完整？

## 第二部分：评分与解释

### 定位

这是为想要深入理解的读者准备的。展示第一部分的判断从何而来——语言模式是什么、场景定位在哪里、归纳信度有多高。

### 写作框架

依次呈现：置信度综合指数 → 关键语言模式观察 → 场景光谱定位 → 情绪轨迹分析 → 人格归纳（中文→大七 / 欧美→大五）→ 依恋归纳（含面具厚度分析）→ 交叉验证 → 可选输出层。

### 结构化输出骨架

以下为第二部分**必须出现的板块**，按顺序排列。每个板块的内容由Step 1→3的数据决定，数据不足的板块标注"不做X推断"而非跳过。

```
板块1：置信度综合指数
必含：综合指数X/10 + 六因子得分 + 结论措辞
格式：使用reports.md定义的置信度表格

板块2：关键语言模式观察
必含：≥2个稳定模式（每个有≥2处原文引用）
如有跨语境数据：追加语境调制模式
格式：每个模式按 ▌模式N：[名称] → 出现在 → 例：格式

板块3：场景光谱定位
必含：≥3个命中场景（定位值+原文引用）
如<3个：标注"场景证据不足"
格式：场景 | 光谱 | 定位 | 原文引用 表格

板块4：情绪轨迹分析（样本≥300字时必含）
必含：轨迹类型 + 关键转折点 + 与Step 3交叉验证
格式：使用reports.md定义的情绪轨迹模板
样本<300字：此板块跳过

板块5：人格归纳
必含：各维度评分条 + 场景证据 + 一致性判断 + 简短分析
模型选择：中文→大七，欧美→大五（不可混用）
每个维度必含：正面贡献+反面代价各一句
格式：使用reports.md定义的人格归纳模板

板块6：依恋方向
如有关系场景数据：输出依恋类型+分析
如无关系场景：标注"不做依恋推断"
勿编造不存在的关系数据

板块7：ND观察层（ND标记≥●中时必含）
格式：使用reports.md定义的ND层模板
ND标记▲弱或无：此板块跳过

板块8：可选输出层（样本≥600字时选含）
```

> 8个板块中，1-6为基础骨架（必出现），7-8为条件骨架（满足条件时出现）。
> 板块3/4/6数据不足时不硬凑，标注"数据不足，不做X推断"。

### 第一部分：关键语言模式观察

从Step 1的扫描结果中提取**最具有区分度的模式**，分为两类呈现。

**① 稳定模式**（跨所有语境一致出现）

```
▌模式1：[模式名称]
出现在：全部28段文字
结构：[句式/标点/语法结构描述]
例：「原文引用1」
    「原文引用2」
    「原文引用3」
→ [此模式指向的心理倾向，带置信度措辞]
```

每个模式必须有至少2处不同语境的原文引用。

**② 语境调制模式**（随语境系统性变化）

```
▌模式N：[模式名称]
语境A：[频率/密度/分布描述]
语境B：[频率/密度/分布描述]
切换条件：[什么语境因素触发变化]
→ [切换行为本身指向什么，而非某一端的数值]
```

### 第六部分：神经多样性特征观察（△ 探索性）

当Step 1.5扫描检测到一个或多个ND标记激活时，在交叉验证后追加神经多样性观察层。本层不产生独立诊断，而是为前述人格/依恋归纳提供调节说明。

```
─── 神经多样性特征观察（△ 探索性）───

本分析层用于调节人格/依恋维度的解读——某些语言模式可能源于
神经类型的认知处理差异，而非人格特质。以下结论基于语言模式，
可能受文化基线校准规则影响（见规则说明）。

高敏感特质（HSP）：X/10  【●中信号】  confound方向：N↗
├─ 模式：感官语言密集（传感器描述N处，跨语境X处）
├─ 例：「原文引用」
└─ 调节建议：N原始评分中可能包含感官超载成分，考虑下调1-2分

ASD谱系特征：X/10  【▲弱信号】  confound方向：E↘  A↗
├─ 模式：系统化加工（规则提取N处）+ masking标记（X处）
├─ 例：「原文引用」
└─ 调节建议：E低分可能包含社交脚本疲劳成分；A偏高可能反映规则坚持

ADHD特质：X/10  【▲弱信号】  confound方向：A↘
├─ 模式：注意力切换（话题跳跃N处）+ 超聚焦描述（X处）
├─ 例：「原文引用」
└─ 调节建议：A低分可能包含执行功能差异成分

阅读障碍相关认知模式：X/10  【△探索性】  confound方向：I↘  O↗
├─ 模式：视觉/空间思维描述 + 替代学习策略
├─ 例：「原文引用」
└─ 调节建议：I评估可能低估了非语言认知能力

综合调节（confound方向 → 实际校准）：

| 维度 | 原始评分 | ND confound | 调节方式 | 校准后评分 |
|------|---------|-------------|---------|-----------|
| N    | X/10    | ND1↑ (N↗)   | 下调1分   | X-1/10 ▲ |
| E    | X/10    | ND4↑ (E↘)   | 上调1分   | X+1/10 ▲ |
| A    | X/10    | ND3↑ (A↘)   | 下调1分   | X-1/10 ▲ |
| O    | X/10    | ND2↑方向一致 | 不调节    | X/10     |

调节说明：
├─ 凡标注confound方向的维度，校准后评分附加▲标记，
│  "该维度信度受ND confound影响"
├─ 当ND调节与一致性调制方向冲突时，以一致性调制为准，
│  ND仅改变置信度标记而非分数
├─ 如有ASD/ADHD中信号以上，依恋中的回避倾向解读需考虑
│  ND社交差异作为替代解释
└─ 隐喻性感官语言（非字面超载）仅做减半调节或不调节

⚠️ 本分析基于语言模式推断，不能替代临床评估
```

### 拼凑样本报告规则

当Input Preparation标记为"拼凑样本"时，报告需做以下调整：

**报告开头标注**：
```
⚠️ 该样本由不同时期/不同对象/不同情景的碎片拼凑而成。
以下结论为跨碎片平均趋势，各维度附碎片间离散度标记。
```

**第一部分调整**：
- 聚焦跨碎片一致出现的核心特征，避免对仅在单个碎片中出现的特征做展开
- 对不一致的特征，呈现为"你在不同时期/面对不同人时差异较大"的张力描述，而非单一判断
- 不出现"你总是……"类绝对化描述

**第二部分调整**：
- 人格归纳各维度评分附碎片离散度标记，格式：
  ```
  ■ N 情绪性    ████████░░ 加权均值:X/10  【离散度:低/中/高】
  碎片评分：[碎片A:8, 碎片B:9, 碎片C:6] | 权重:[A:0.4, B:0.35, C:0.25] | 加权均值:7.7 | 加权标准差:1.1
  ```
- 依恋风格：仅当≥2个关系碎片指向一致时输出，否则标注"关系场景碎片不一致，不做依恋推断"
- 情绪轨迹分析：跳过整体轨迹曲线，改做各碎片独立效价标记后取总体效价均值

**第二部分人格归纳（拼凑样本精简版）**：
当拼凑样本字数在300-600字区间时，以精简评分表替代完整人格归纳：

```
─── 拼凑样本 · 人格均值归纳（精简）───

各维度跨碎片加权均值与离散度：
■ N 情绪性    ████████░░  加权均值:X/10  【离散度:低/中/高】  
● O 开放性    ██████░░░░  加权均值:X/10  【离散度:低/中/高】
[etc.]

离散度说明：
- 低离散（标准差≤1）：跨碎片一致，保持置信度
- 中离散（1<标准差≤2）：跨碎片有差异，置信度降一级
- 高离散（标准差>2）：跨碎片分歧大，结论仅供参考

依恋归纳：[一致输出 / 关系碎片不一致，不做推断]
```
